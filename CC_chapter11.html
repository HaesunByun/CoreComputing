Chapter 11. 그래프#

학습목표와 기대효과

  • 학습목표

    • 그래프의 개념을 이해한다.

    • 그래프를 표현하는 2가지 방법을 이해한다.

    • 그래프의 탐색 방법을 이해한다.

  • 기대효과

    • 그래프 탐색을 이용한 문제해결 능력을 배양할 수 있다.

그래프는 요소들이 서로 복잡하게 연결되어 있는 관계를 표현하는 자료구조이다. 많은 역들이 어떻게 연결되어 있는지를 보여주는 지하철 노선도, 사람들의 복잡한 친구 관계를 보여주는 소셜네트워크 인맥지도, 어떤 지역에서 다른 지역으로 가는 가장 가까운 경로를 표현하는 경로 찾기 등이 그 예이다.

그래프는 수학자 **오일러(Euler)**에 의해 처음 창안되었다. 오일러의 그래프 이론에서는 쾨니히스베르크의 다리 문제가 유명하다.

쾨니히스베르크에는 프레겔강이 흐르고 있고, 이 강에는 큰 섬이 있다. 그리고 이 섬과 도시의 나머지 부분을 연결하는 7개의 다리가 있다. 이때 7개의 다리들을 한 번만 건너면서 처음 시작한 위치로 돌아오는 길이 있는가 하는 것이 문제이다. 오일러는 모든 정점에 연결된 간선의 개수가 짝수일 때만 가능하다고 오일러의 정리를 증명하였다.

나무위키

여기에서는 그래프의 표현방법과 기본 알고리즘을 살펴보자.

그래프 용어#

  • 정점(Vertex 또는 Node): 데이터를 저장하는 위치

  • 간선(Edge): 정점를 연결하는 선. 링크(Link) 또는 브랜치(branch) 로도 불린다.

  - 하나의 간선은 두 개의 정점을 연결한다.
  - 그래프 G는 G=(V,E)로 표현, V는 정점의 집합이며, E는 간선의 집합이다.
  - 정점 A와 B를 연결하는 간선을 (A,B)로 표현한다.
  - 정점 A와 B로 간선의 방향이 있는 경우 <A,B>로 표현한다.

  • 방향 그래프(Directed Graph): 간선에 방향이 있는 그래프

- **무방향 그래프(Undirected Graph)**: 간선에 방향이 없는 그래프

  • 예) 그래프 G1

    V(G1) = [0,1,2,3]

    E(G1) = [(0,1),(0,2),(0,3),(1,2),(1,3),(2,3)]

  • 예) 그래프 G2

    V(G2) = [0,1,2]

    E(G2) = [<0,1>, <1,0>, <1,2>]

  • 예) 그래프 G3

    V(G3) = [0,1,2,3,4,5,6,7]

    E(G3) = [(0,1),(0,2),(0,3),(1,2),(1,3),(2,3),(4,5),(5,6),(6,7)]

  • 차수(Degree): 정점에 인접한 정점의 수

    • 방향 그래프에서는 정점으로 들어오는 차수를 진입 차수(In-degree), 정점에서 나가는 차수를 진출 차수(Out-degree)로 구분한다.

    • 방향 그래프는 진입 차수와 진출 차수가 다를 수 있다. <\(v_i, v_j\)> \(\neq\) <\(v_j, v_i\)>

    • 무방향 그래프는 <\(v_i, v_j\)> = <\(v_j, v_i\)>이다.

  • 경로(Path): 시작 정점 \(u\)부터 도착 정점 \(v\)까지의 정점들을 나열한 표현한다.

    • 단순경로: 경로 상의 정점들이 모두 다른 경로

    • [A,B,D,C]: 시작 정점 A로부터 도착 정점 C까지의 단순 경로

  • 싸이클(Cycle): 시작 정점과 도착 정점이 동일한 단순 경로

    • [A,B,D,C,A]

  • 완전 그래프(Complete graph): 그래프에서 간선의 수가 최대인 그래프

    • 무방향 그래프의 경우 \(n\)개의 정점이 있다면 간선의 수는 \(n(n-1)/2\)

      • (예)정점이 4개인 그래프에서 간선의 최대 개수는 6

    • 방향 그래프의 경우 \(n\)개의 정점이 있다면 간선의 수는 \(n(n-1)\)

  • 인접(adjacent): 무방향 그래프에서 정점 a,b에 대하여 간선(a,b)가 있으면, 정점 a는 정점 b에 인접한다고 한다.

  • 부속(incident): 무방향 그래프에서 정점 a,b에 대하여 간선(a,b)가 있으면, 간선(a,b)는 정점 a와 b에 부속한다고 한다.

  • 가중치(Weighted) 그래프: 간선에 가중치가 부여된 그래프

    • 가중치는 두 정점 사이의 거리, 시간 등이 될 수 있음

  • 트리(Tree): 싸이클이 없는 그래프

  • 신장트리(Spanning Tree): 그래프의 모든 정점들을 싸이클없이 연결하는 부분그래프

그래프 표현 방법#

  • 그래프를 자료구조로 저장하고 표현하는 방법으로는 인접 행렬(Adjacency Matrix)와 인접 리스트(Adjacency List)가 있다.

그래프 탐색 알고리즘#

  • 그래프 탐색은 하나의 정점에서 시작하여 모든 정점을 한 번씩 방문하는 작업이다.

    • 예, 전자회로 연결 판단, 미로 출구 찾기

  • 대표적인 그래프 탐색 알고리즘은 **깊이 우선 탐색(DFS, depth first search)**와 너비 우선 탐색(BFS, breath first search) 알고리즘이 있다.

깊이 우선 탐색(DFS, depth first search)#

  • 그래프에서 깊이 탐색을 우선적으로 탐색하는 알고리즘

  • 동작과정

    1. 임의의 정점에서 시작하여 이웃하는 하나의 정점을 방문한다. 이때 방문한 정점은 반드시 방문되었다고 표시를 해야 한다.

    2. 방금 방문한 정점의 이웃 정점을 방문하며, 이웃하는 정점을 모두 방문한 경우에는 이전 정점으로 되돌아 간다.

    3. 거기서 다시 아직 방문하지 않은 인접 정점을 찾아 동일한 방법의 탐색을 수행하는 방식이다.

  • 스택이나 재귀함수를 이용하여 구현 가능하다.

    1. 탐색 시작 노드를 스택에 삽입하고 방문 처리

    2. 스택의 최상단 노드에 방문하지 않은 인접한 노드가 하나라도 있으면 그 노드를 스택에 넣고 방문 처리.방문하지 않은 인접 노드가 없으면 스택에서 최상단 노드를 꺼냄

    3. 더 이상 2번의 과정을 수행할 수 없을 때까지 반복

  • Step 0: 그래프를 준비(방문 기준: 번호가 낮은 인접 노드부터)

    • 시작 노드: 1

  • Step 1: 시작 노드인 ‘1’를 스택에 삽입하고 방문 처리

  • Step 2: ‘1’의 인접 노드 ‘2’, ‘3’, ‘8’ 가운데 가장 낮은 번호를 가진 노드 ‘2’를 스택에 삽입하고 방문 처리

  • Step 3: ‘2’의 인접 노드 ‘7’이 있음. 방문하지 않은 노드이므로 ‘7’를 스택에 삽입하고 방문 처리

  • Step 4: ‘7’의 방문하지 않은 인접 노드 ‘6’, ‘8’이 있음. 이 가운데 가장 낮은 번호를 가진 방문하지 않은 노드 ‘6’을 스택에 삽입하고 방문 처리

  • Step 5: 스택의 최상단 노드인 ‘6’에 방문하지 않은 인접 노드가 없음. 따라서 스택에서 ‘6’번 노드를 꺼냄

  • Step 6: 스택의 최상단 노드인 ‘7’에 방문하지 않은 인접 노드 ‘8’이 있음. 따라서 ‘8’번 노드를 스택에 넣고 방문 처리

  • 이러한 과정을 반복하였을 때 전체 노드의 탐색 순서(스택에 들어간 순서)가 된다.

DFS 알고리즘 재귀적 구현#

def dfs_recursive(graph, start, visited=None):
    if visited is None:
        visited = set()
    visited.add(start)
    print(start, end=' ')

    for neighbor in graph[start]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited)

너비 우선 탐색(BFS, breath first search)#

  • 그래프에서 가까운 노드부터 우선적으로 탐색하는 알고리즘

  • 시작 정점으로부터 가까운 정점을 먼저 방문하고 멀리 떨어져 있는 정점을 나중에 방문하는 순회 방법이다.

  • 가까운 거리에 있는 정점들을 차례대로 저장하고, 들어간 순서대로 꺼낼 수 있는 자료구조인 큐를 이용하여 구현이 가능하다.

  • 동작과정

    1. 탐색 시작 노드를 큐에 삽입하고 방문처리

    2. 큐에서 노드를 꺼낸 뒤에 해당 노드의 인접 노드 중에서 방문하지 않은 노드를 모두 큐에 삽입하고 방문처리

    3. 더 이상 2번의 과정을 수행할 수 없을 때까지 반복

  • Step 0: 그래프를 준비(방문 기준: 번호가 낮은 인접 노드부터)

    • 시작 노드: 1

  • Step 1: 시작 노드인 ‘1’를 큐에 삽입하고 방문 처리

  • Step 2: 큐에서 노드 ‘1’을 꺼내 방문하지 않은 인접 노드 ‘2’,’3’,’8’을 큐에 삽입하고 방문 처리

  • Step 3: 큐에서 노드 ‘2’를 꺼내 방문하지 않은 인접 노드 ‘7’을 큐에 삽입하고 방문 처리

  • Step 4: 큐에서 노드 ‘3’을 꺼내 방문하지 않은 인접 노드 ‘4’, ‘5’를 큐에 삽입하고 방문 처리

  • Step 5: 큐에서 노드 ‘8’을 꺼내고 방문하지 않은 인접 노드가 없으므로 무시

  • 이러한 과정을 반복하여 전체 노드의 탐색 순서(큐에 들어간 순서)대로 방문

그래프 탐색의 응용#

  • 하이퍼링크로 연결 된 사이트들 간의 관계를 방향 그래프로 표현하였을 때, 간선을 통해 사이트에서 사이트로의 영향력을 직간접적으로 분석할 수 있다.

  • 하이퍼링크를 통해 사이트의 영향력이나 중요도를 판단하는 방법을 페이지랭크(PageRank)라 한다.

  • 그림을 보면, 두 사이트 간의 하이퍼링크를 통해 영향력이 전달되는 것을 볼 수 있고, 이러한 관계가 복잡해지면서 전체 사이트들 간의 중요도를 크기로 표현하고 있다.

  • 간선의 흐름에 따라 노드를 추적해보면 거리가 멀어질수록 영향력이 떨어지는 것을 알 수 있다.

스택을 이용한 DFS 크롤링 예제#

from requests import request
from requests.compat import urlparse, urljoin
from bs4 import BeautifulSoup

# 시작 URL 설정
seed = 'https://haesunbyun.github.io/Core-Computing-Ver.2/intro.html'
base = urlparse(seed)[1]  # 기본 도메인 추출
tovisit = [seed]  # 방문할 URL 리스트 초기화, 스택
visited = []  # 방문한 URL 리스트 초기화

while tovisit:
    url = tovisit.pop()  # 방문할 URL 리스트에서 마지막 URL을 가져옴
    page = request('get', url)  # 해당 URL에 GET 요청을 보냄
    soup = BeautifulSoup(page.content, 'lxml')  # 응답 내용을 BeautifulSoup을 사용해 파싱
    visited.append(url)  # 현재 URL을 방문한 URL 리스트에 추가

    for _ in soup.select('a[href]'):  # 페이지 내의 모든 링크를 순회
        new_url = urljoin(seed, _['href'])  # 절대 URL로 변환
        # 링크가 페이지 내부 링크이거나 같은 도메인의 링크가 아닌 경우 건너뜀
        if _['href'][0] == '#' or urlparse(new_url)[1] != base:
            continue
        # 새로운 URL이 방문 예정 또는 방문한 URL 리스트에 없는 경우에만 추가
        if new_url not in visited + tovisit + [seed+'/']:
            tovisit.append(new_url) # 새 URL을 방문할 URL 리스트에 추가

# 방문한 URL 리스트 출력
visited

스택을 이용한 DFS 시 탐색범위 제한#

  • 뉴스 사이트처럼 문서들이 관련 주제에 따라 계층형 관계로 구성되어 있을 때, 특정 링크를 따라 계속해서 문서들을 추적하다 보면 결국 원래의 주제에서 멀리 떨어진 내용들을 다루게 된다.

  • 따라서 탐색 범위 제한을 통해 무한히 확장하는 인터넷 공간상에서 사이클에 빠지지 않도록 **비 순환성(Acyclic)**으로 만들거나, 시드로부터 너무 멀리 떨어져 있어서 발생할 수 있는 내용적 비 유사성을 사전에 방지하고 크롤러가 적당한 시점에 탈출 할 수 있도록 참조 범위를 기준으로 제한할 필요가 있다.

  • 아래의 코드는 스택을 이용한 깊이 우선 탐색 시, 시드를 0부터 시작하여 하이퍼링크를 통해 다음 페이지를 탐색할 때마다 순차적으로 1씩 증가하였고, 최대 참조 범위가 2 미만일 때 까지 만으로 제한한 예이다.

from requests import request
from requests.compat import urlparse, urljoin
from bs4 import BeautifulSoup

# 시작 URL 설정
seed = 'https://news.naver.com/'
base = urlparse(seed)[1]  # 기본 도메인 추출
tovisit = [(seed,0)]  # 방문할 URL 리스트 초기화
visited = []  # 방문한 URL 리스트 초기화

while tovisit:
    url = tovisit.pop(-1)  # 방문할 URL 리스트에서 첫 번째 URL을 가져옴
    page = request('get', url[0])  # 해당 URL에 GET 요청을 보냄
    soup = BeautifulSoup(page.content, 'lxml')  # 응답 내용을 BeautifulSoup을 사용해 lxml문서로 파싱
    visited.append(url)  # 현재 URL을 방문한 URL 리스트에 추가

    for _ in soup.select('a[href]'):  # 페이지 내의 모든 링크를 순회
        new_url = urljoin(seed, _['href'])  # 절대 URL로 변환

        # 링크가 비어있거나, 링크가 페이지 내부 링크이거나 같은 도메인의 링크가 아닌 경우 건너뜀
        if not _['href'] or _['href'][0] == '#' or new_url[:4] != 'http' or url[1] > 1:
          continue
        # 새로운 URL을 방문 예정 또는 방문한 URL 리스트에 없는 경우에만 추가
        if new_url not in [_[0] for _ in visited] + [_[0] for _ in tovisit] + [seed + '/']:
          tovisit.append((new_url, url[1]+1))

    # 오래 걸려서 10개까지만 제한
    if len(visited)>=10:
      break

visited

큐를 이용한 BFS 시 탐색범위 제한#

  • 탐색한 도메인을 지정함으로서 탐색 범위를 제한한 예이다.

from requests import request
from requests.compat import urlparse, urljoin
from bs4 import BeautifulSoup

# 시작 URL 설정
seed = 'https://haesunbyun.github.io/Core-Computing-Ver.2/intro.html'
seed = 'https://news.naver.com/'
base = urlparse(seed)[1]  # 기본 도메인 추출
tovisit = [seed]  # 방문할 URL 리스트 초기화
visited = []  # 방문한 URL 리스트 초기화
enabled= ['_sources'] # 탐색을 허용하는 도메인 서브

while tovisit:
    url = tovisit.pop(0)  # 방문할 URL 리스트에서 첫 번째 URL을 가져옴
    page = request('get', url)  # 해당 URL에 GET 요청을 보냄
    soup = BeautifulSoup(page.content, 'lxml')  # 응답 내용을 BeautifulSoup을 사용해 lxml문서로 파싱
    visited.append(url)  # 현재 URL을 방문한 URL 리스트에 추가

    for _ in soup.select('a[href]'):  # 페이지 내의 모든 링크를 순회
        new_url = urljoin(seed, _['href'])  # 절대 URL로 변환

        # 링크가 비어있거나, 링크가 페이지 내부 링크이거나 같은 도메인의 링크가 아닌 경우 건너뜀
        if _['href'][0] == '#':
          continue
        if True not in map(lambda x:x in enabled, urlparse(new_url)[2].split('/')):
          continue
        # 새로운 URL을 방문 예정 또는 방문한 URL 리스트에 없는 경우에만 추가
        if new_url not in visited + tovisit + [seed + '/']:
          tovisit.append(new_url)

마무리#

  • DFS: 주어진 시작 정점에서 시작하여 가능한 깊이까지 탐색한 후, 더 이상 갈 수 없을 때 다른 경로로 되돌아가 탐색을 이어가는 방식이다.

  • BFS: 주어진 시작 정점에서 시작하여 인접한 모든 정점을 먼저 탐색한 후, 그 다음 단계의 정점을 탐색하는 방식이다.
